### 組織管理

**`組織 ＞ プロジェクト ＞ チーム`**  
のような関係性です

- **Organization（組織）**
    - 会社単位とかで設定
- **Project（プロジェクト）**
    - 複数のプロジェクト作成可能
    - 1つのプロジェクト内に複数のTeamとAreaを作成できる
- **Team（チーム）**
    - 1つのプロジェクトに属する（複数のプロジェクトをまたげない）
    - 複製作成可能
    - さらにサブチームも作成可能
    - チームごとに1つずつBoardsを管理できる
- **Area（エリア）**
    - プロジェクト、チーム、作業項目、ボードに紐づけられるタグのようなもの
    - 1つのプロジェクトに属する（複数のプロジェクトをまたげない）
    - 複製作成可能
    - ツリー上に作成可能
    - 例えば、チームとWork Items（作業項目）を紐づけるときにAreaを使用します
      - チームA → エリアパス `Project/Frontend`
      - 作業項目1: エリアパス `Project/Frontend` → チームAに表示

## Azure Boards

- **Work Items（作業項目）**: タスク
  - Bug（バグ）
  - Task（タスク）
  - User Story（ユーザーストーリー）
  - Epic（エピック）
- **Boards（ボード）**: viewみたいなもの
   - Boards（ボード）
   - Backlog（バックログ）
   - Kanban Board（カンバンボード）
   - Sprints（スプリントビュー）
   - Queries（クエリ）
   - Burndown Chart / Burnup Chart（バーンダウンチャート / バーンアップチャート）
   - Delivery Plans（タイムライン）

## アカウントの紐づけ

- **Organization（組織）レベル**
    - ユーザーを組織に招待する必要がある
    - 組織内のプロジェクト単位で権限を制限することが可能
- **Project（プロジェクト）レベル**
    - 組織内のユーザーを紐づけられる
    - プロジェクトのチームに所属しなくてもプロジェクト内のリポジトリ、パイプライン、作業項目などにアクセスすることが可能
- **Team（チーム）レベル**
    - 組織内のユーザーを紐づけられる
    - 特定のエリア（Area）やスプリント（Iteration）など、チーム固有のリソースやビューにアクセスできるようになる
    - そのチームが担当するバックログやボードにも紐づく

## 認証方法（サインイン）

- **Organization（組織）ごとに設定可能**
    - **Microsoft アカウント：**
        - 個人のMSアカウントを使用してログインする方法
    - **Microsoft Entra ID に関連付けられたアカウント （旧 Azure Active Directory）：**
        - 組織のEntra IDに接続して組織内のユーザーと連携する方法
